# Alert Playbooks

This directory contains playbooks for responding to alerts generated by the GitHub Metrics Aggregator system.

## Purpose
To provide actionable instructions for on-call engineers to diagnose and resolve specific alert conditions.

## Files

- **`BadRequests.md`**: Playbook for responding to a high rate of 4xx HTTP responses from the webhook service.
- **`ContainerUsage.md`**: Playbook for responding to high CPU or memory usage in Cloud Run containers.
- **`ForwardProgressFailed.md`**: Playbook for responding to failures in the retry job or data pipeline that prevent new events from being processed.
- **`RequestLatency.md`**: Playbook for responding to high latency in handling webhook requests.
- **`ServerFaults.md`**: Playbook for responding to a high rate of 5xx HTTP responses from the services.

## Usage
These files are intended to be linked from alert notifications (e.g., in Cloud Monitoring) so that engineers can quickly access them when paged.
